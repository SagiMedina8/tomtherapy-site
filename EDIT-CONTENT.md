# עריכת תוכן האתר

התיקייה הזו מכילה את **כל הטקסטים** שמופיעים באתר. כדי לשנות תוכן — ערוך את קובץ ה-JSON המתאים. לא צריך לגעת בקוד.

## איזה קובץ אחראי על איזה דף

| קובץ              | דף באתר               | מה מכיל                                          |
|-------------------|-----------------------|--------------------------------------------------|
| `brand.json`      | כל האתר (גלובלי)      | שם, טלפון, אימייל, כתובות, שעות, תפריט ניווט     |
| `home.json`       | `/` (עמוד הבית)       | hero + הסקשן "הגוף מדבר בשפה שלו"                |
| `about.json`      | `/about`              | כותרת, פסקאות, ה-CTA בתחתית                      |
| `services.json`   | `/services`           | 4 כרטיסי שירות + כותרת + CTA bar                |
| `process.json`    | `/process`            | 4 שלבי התהליך + כותרת + CTA                      |
| `testimonials.json` | `/testimonials`     | רשימת ההמלצות + כותרת                            |
| `contact.json`    | `/contact`            | כותרות, טופס יצירת קשר, תוויות                    |
| `booking.json`    | `/booking`            | כותרת + URL של Plannie                           |
| `articleIndex.json` | `/articles`         | כותרת עמוד האינדקס של המאמרים                    |
| `bodyMap.json`    | `/` (בודי-מאפ)        | תוכן 14 אזורי הגוף לתצוגה האינטראקטיבית          |

**את תוכן המאמרים עצמם** (3 כתבות) עורכים ב-`src/content/articles/*.md` — לא כאן.

---

## איך עורכים ב-GitHub (Web UI)

1. היכנס ל-GitHub ונווט לקובץ — למשל `src/content/site/home.json`.
2. לחץ על אייקון העיפרון (✎) בצד ימין למעלה — "Edit this file".
3. ערוך את הערכים שאתה רוצה. **רק את מה שאחרי הנקודתיים** `:` — אל תשנה את שמות המפתחות (מה שלפני הנקודתיים).
4. גלול למטה. ב-"Commit changes" תן הסבר קצר בעברית (למשל "עדכון ניסוח של כותרת הבית") ולחץ "Commit changes".
5. תוך דקה-שתיים Netlify יבנה מחדש את האתר. השינוי יופיע אוטומטית.

## חוקי עריכה חשובים

### ✅ מותר

- לשנות כל **ערך** (כל דבר שבתוך מרכאות כפולות אחרי `:`)
- להוסיף מילה, להוריד מילה, לשכתב משפט שלם
- לכתוב Latin words בתוך עברית — יעבוד אוטומטית (דוגמה: `"טיפול Dry Needling מתקדם"`)

### ❌ אסור

- לשנות את ה**שמות של השדות** (המפתחות לפני ה-`:`). למשל אל תשנה `"headline"` ל-`"headlines"` — זה ישבור את האתר.
- למחוק שדות שלמים
- למחוק מרכאות, נקודתיים `:`, או פסיקים `,` — הם חלק ממבנה הקובץ ואם ימחקו האתר לא יקומפל.

## איך לעשות שורה חדשה בטקסט

בתוך ערך של מחרוזת, שים `\n` איפה שאתה רוצה שורה חדשה:

```json
"paragraph1": "שורה ראשונה.\nשורה שנייה."
```

**אבל שים לב:** חלק מהטקסטים באתר מורכבים משני שדות נפרדים (למשל `headlineLine1` ו-`headlineLine2`), אז אתה לא צריך `\n` — פשוט ערוך כל שורה בנפרד.

---

## איך להכניס Latin word בתוך טקסט עברי

פשוט לכתוב — יעבוד אוטומטית. הדפדפן יודע לזהות Latin characters בתוך טקסט עברי וינהל את הכיווניות לבד.

```json
"description": "עבודה ממוקדת על Fascia ועל הידבקויות רקמת חיבור."
```

## דוגמה מלאה: לפני ואחרי

**בקובץ `home.json`, לפני:**
```json
"hero": {
  "eyebrow": "SPORT THERAPY · TOM KUSHLIN",
  "headlineLine1": "הכאב לא חייב",
  "headlineLine2": "להיות חלק מהחיים",
  ...
}
```

**אחרי עריכה (שיניתי את שתי שורות הכותרת):**
```json
"hero": {
  "eyebrow": "SPORT THERAPY · TOM KUSHLIN",
  "headlineLine1": "יש דרך אחרת",
  "headlineLine2": "להשתחרר מהכאב",
  ...
}
```

שים לב: המפתחות (`eyebrow`, `headlineLine1`, `headlineLine2`) **לא השתנו**. רק הערכים שלהם.

## נפלתי על שגיאה — מה לעשות?

1. אם ה-commit נכשל ב-GitHub — קרא את הודעת השגיאה. בדרך כלל הסיבה היא שגיאת JSON (למשל מרכאה שנמחקה, או פסיק מיותר).
2. אם השינוי עבר commit אבל Netlify מראה "build failed" — זו אותה בעיה. פתח את הקובץ, השווה לגרסה קודמת דרך History של GitHub, ותקן.
3. במקרה חירום — נווט ל-History של הקובץ ב-GitHub (הלחצן `History`), בחר את הגרסה האחרונה שעבדה, ולחץ `Restore`.

אם אתה לא בטוח — תשאל לפני שאתה עושה commit.
